# GreenLife Sustainable Fashion Website

A modern, responsive e-commerce website for GreenLife - a sustainable fashion brand, built using The Cube Method Framework.

## The Cube Method Implementation

This website implements all three layers of The Cube Method Framework to ensure strategic alignment between business goals and digital presence.

### 1. Transactional Components (The "Action" Layer)

Features that enable immediate, concrete user actions:

- **User Registration & Login System** - Allows users to create accounts and log in
- **Secure Checkout Process** - SSL-certified payment processing
- **Contact Forms** - Newsletter signup and customer support forms
- **Integration Points** - Social media connections and CRM integrations
- **Security Badges** - Visual indicators of security compliance

Located primarily in:
- Header authentication buttons
- Modal forms
- Newsletter subscription section

### 2. Transformational Components (The "Behavior" Layer)

Features designed to change user behavior and build engagement:

- **Educational Blog** - Resources on sustainable fashion and conscious consumption
- **Impact Dashboard** - Shows personal environmental contributions
- **Community Forum** - Platform for user interaction and discussion
- **Training Materials** - Guides for using the platform and understanding sustainability
- **Analytics Reports** - Data visualization for informed decision-making

Located in:
- Transformational Components section
- Blog and resource pages (accessible through cards)

### 3. Transcendent Components (The "Vision" Layer)

Features that communicate brand values and long-term vision:

- **Mission & Vision Statement** - Core company values and goals
- **Sustainability Stats** - Environmental impact metrics
- **Testimonials** - Customer success stories
- **Brand Story** - Narrative connecting customers to the brand's purpose
- **Value Proposition** - Clear articulation of benefits beyond products

Located in:
- Hero section
- Transcendent Components section
- Testimonials section
- Footer mission statement

## Deployment

Simply upload all files to your web server or hosting provider. No build process required.

## Customization

To customize the website:

1. Modify colors in `:root` variables in `css/styles.css`
2. Replace images in the `assets/` folder
3. Update content in `index.html`
4. Adjust functionality in `js/script.js`
